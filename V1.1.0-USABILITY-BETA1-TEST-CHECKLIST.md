# v1.1.0-usability beta 1 hardware checklist

This beta targets the Waveshare ESP32-S3-LCD-2 and WeebLabs DSPi `release/v1.1.5`. Complete this checklist before promoting the branch or replacing the stable v1.0.0 release.

## 1. Boot and link

- [ ] Power-cycle DSPi and the front panel from a cold start.
- [ ] Confirm the panel reaches Home and reports a valid source, preset and sample rate.
- [ ] Open `System > Status`; confirm DSPi, UART, remote and front-panel version information is readable.
- [ ] Confirm Status does not contain a special Psy Bass capability line.

## 2. Listening controls and Console synchronisation

- [ ] Toggle Loudness, Crossfeed, Leveller and Psy Bass from the panel and confirm each change in DSPi Console.
- [ ] Change the same features in Console and confirm the panel follows them.
- [ ] Verify all editable parameters still wrap or clamp at their original safe limits.
- [ ] Verify UART notifications update the panel promptly and the 1.2-second fallback polling still recovers missed changes.

## 3. Save Current preset

Use a non-critical test slot or back up the active DSPi preset first. `Save Current` deliberately overwrites the active slot.

- [ ] Load a named preset and note its slot and name.
- [ ] Change several DSP features, user volume and input source.
- [ ] Choose `Preset > Save Current`; verify the confirmation defaults to `No`.
- [ ] Select `No`; power-cycle and confirm no overwrite occurred.
- [ ] Repeat, select `Yes`, and wait for `Preset saved` or `Save accepted`.
- [ ] Load another preset and then reload the saved slot. Confirm DSP features, user volume, input source and name return from DSPi.
- [ ] Power-cycle and confirm DSPi startup behavior is unchanged and the saved slot remains valid in Console.

## 4. Global Volume Limit

- [ ] Set `System > Volume Limit` to a safe test value and apply it.
- [ ] Confirm `Global limit saved` appears.
- [ ] Switch through several presets; confirm the limit does not change.
- [ ] Power-cycle DSPi; confirm the same limit returns.
- [ ] In Console, confirm Master Volume persistence mode is `Independent`/global after applying a limit from the panel.

## 5. Preset and source distance views

- [ ] From Home, use Previous/Next preset shortcuts and confirm the large user name appears with a small `P1`-style slot underneath and no `Preset` label.
- [ ] Confirm long preset names scale without clipping.
- [ ] Use Previous/Next/Cycle input shortcuts and confirm the large source screen appears for every available source.
- [ ] Confirm unavailable optional S/PDIF and ADAT inputs are skipped.
- [ ] Confirm the overlay returns to Home after its timeout and a new key press dismisses it immediately while still processing that key.

## 6. Five Home shortcuts and migration

- [ ] With preserved v1.0.0 settings, confirm Up/Down/Left/Right assignments survive the upgrade.
- [ ] Confirm Centre defaults to `Open Menu`.
- [ ] Assign and test Previous/Next preset, Previous/Next input, Cycle input, VU, Mute and feature toggles.
- [ ] Confirm D-pad keys retain normal navigation inside menus.

## 7. Encoder behavior

- [ ] Confirm two physical notches move one menu item and direction changes do not cause a stray step.
- [ ] Confirm one notch still changes Home and VU user volume by 0.5 dB.
- [ ] Confirm one physical notch changes an edited toggle or value by exactly one option step (for example 5%, not 10%).
- [ ] Confirm short press, mute hold and five-second VU hold retain their existing behavior.

## 8. Screen power settings

- [ ] Test Off, 30 seconds, 1 minute, 3 minutes and 5 minutes.
- [ ] Test `Screen Off` and every dim level from 10% through 80%.
- [ ] Confirm timeout fades to the selected level without blocking UART, BLE or meter service.
- [ ] Confirm encoder rotation, encoder press and every remote key wake the backlight smoothly.
- [ ] Power-cycle and confirm timeout, dim level and brightness persist.

## 9. Meter regression

- [ ] Confirm the stereo L/R segmented meter is visually and behaviorally unchanged.
- [ ] Confirm the analogue meter responds at normal listening levels and can travel above the old -10 VU/50% region on strong peaks.
- [ ] Confirm only the analogue display mapping changed; output clip still drives +5 VU.
- [ ] Confirm mute, stopped USB playback, source changes, preset changes, S/PDIF loss and ADAT loss clear stale readings as before.

## 10. BLE remote regression

- [ ] Confirm the saved remote reconnects after restart.
- [ ] Confirm all 15 learned key roles still work.
- [ ] Confirm a saved remote must be removed before `Find Remote` becomes available.
- [ ] Confirm removing a remote clears its bond/profile/key map but preserves Home shortcut choices.
